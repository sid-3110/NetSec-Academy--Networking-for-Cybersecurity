01 — INTRODUCTION TO NETWORKING
Understand the fundamental concepts of how devices communicate in a network — the bedrock of all cybersecurity knowledge.
📖 Definition

A network is a collection of two or more computing devices connected together to share resources, data, and services. Think of it like a postal system — every device has an address, messages (data) are packaged, routed, and delivered.

💡 Real World Example

When you stream YouTube, your laptop (device 1) sends a request to Google's server (device 2) over a network. The video travels through multiple routers across the internet — each one reading your device's "address" and forwarding data closer to you. That entire process is networking.

Why Networking Matters in Cybersecurity

Every cyberattack travels over a network. SQL injection, phishing, ransomware, DDoS — they all use networking protocols. If you don't understand how data moves, you cannot understand how it's attacked or defended.

🔴

Offensive Security

Pentesting requires port scanning, packet sniffing, and exploiting network-level vulnerabilities.

🔵

Defensive Security

SOC analysts read network logs, detect anomalies, and trace attack paths through the network.

🕵️

Forensics

Incident responders reconstruct attacks from network traffic captures and firewall logs.

☁️

Cloud Security

Cloud misconfigurations expose network services — understanding ports and IPs is essential.

02 — HOW THE INTERNET WORKS
Trace the full journey of data from your browser to a web server and back — understanding every step.
📖 Definition

The Internet is a global system of interconnected computer networks that use the TCP/IP protocol suite to communicate. It's not owned by any single entity — it's a massive, decentralized collection of networks that all agree to speak the same language.

The Client–Server Model
💡 Analogy — Restaurant

You are the client (customer). The website's server is the server (kitchen). Your browser sends a request (order). The server sends back a response (food). The network is the waiter carrying messages between you.

ANIMATED Client–Server Model — Simulate HTTP Request
What Happens When You Visit a Website

This is the complete step-by-step flow — understanding this is essential for both web security and network analysis.

1

You type "google.com" and press Enter

Your browser needs to find Google's IP address. It can't route to a domain name directly — it needs the numeric IP. It first checks its local cache.

2

DNS Resolution — Domain → IP Address

Browser asks a DNS resolver (usually your ISP's or 8.8.8.8): "What is the IP for google.com?" The resolver returns 142.250.80.46. Now your browser knows where to connect.

3

TCP Connection — Three-Way Handshake

Your browser connects to port 443 (HTTPS) on Google's server. A TCP handshake happens: SYN → SYN-ACK → ACK. Connection established.

4

TLS Handshake — Encryption Setup

For HTTPS, a TLS handshake negotiates encryption. Server sends its certificate. Your browser verifies it's legitimate and sets up an encrypted tunnel.

5

HTTP Request Sent

Browser sends: GET / HTTP/1.1 — requesting the homepage. Headers include your browser type, accepted languages, cookies, etc.

6

Server Processes and Responds

Google's server receives the request, generates the response (HTML, CSS, JavaScript, images), and sends it back with status code 200 OK.

7

Browser Renders the Page

Your browser parses HTML, downloads linked resources (images, scripts), runs JavaScript, and displays the page. The entire process takes milliseconds.

ANIMATED Complete Flow: Browser → Google.com (7 Steps)
⚠️ Attacker Insight

Every one of these 7 steps is a potential attack point. DNS spoofing redirects you to a fake site. Man-in-the-Middle attacks intercept traffic between steps 3–5. Understanding the normal flow helps you spot the abnormal — which is exactly what security analysts do.

Every device needs an address. Learn the complete IP addressing system — from IPv4 structure to subnetting.
📖 Definition

An IP address (Internet Protocol address) is a unique numerical label assigned to every device on a network. It serves two functions: identifying the device (who) and providing its location on the network (where) — like a combination of your name and home address.

Subnet Mask and CIDR Notation
📖 What is Subnetting?

A subnet mask divides an IP address into the network portion (which subnet you're on) and the host portion (which specific device). CIDR notation (e.g., /24) is a shorthand that tells you how many bits are used for the network portion.

Static vs Dynamic IP
🔒

Static IP

Manually configured. Never changes. Used for servers, printers, cameras. Easier to firewall-rule — attackers know exactly where to find it.

🔄

Dynamic IP (DHCP)

Automatically assigned by DHCP server. Changes periodically. Used for workstations and phones. DHCP starvation is a real attack.

What is an IPv4 address and how is it structured?
IPv4 is a 32-bit address written as four octets (0-255) separated by dots, e.g., 192.168.1.100. It has two parts: network portion (identifies the subnet) and host portion (identifies the specific device).
👆 Click to reveal answer
What are private IP ranges?
RFC 1918 defines three private ranges not routable on the internet: Class A 10.0.0.0/8, Class B 172.16.0.0/12, Class C 192.168.0.0/16. These require NAT to access the internet.
👆 Click to reveal answer
What does /24 mean in CIDR notation?
/24 means 24 bits are used for the network portion, leaving 8 bits for hosts. Subnet mask is 255.255.255.0, allowing 254 usable host addresses (256 minus network address and broadcast).
👆 Click to reveal answer
What is 127.0.0.1?
127.0.0.1 is the loopback address, commonly called 'localhost'. It refers to the device itself — traffic sent to this address never leaves the machine. Used for testing local services.
👆 Click to reveal answer
Why do attackers care about subnetting?
Subnets define the attack surface. By identifying subnet ranges (e.g., via CIDR), attackers can map which IPs to scan for live hosts. Network segmentation uses subnets to isolate sensitive systems and limit lateral movement.

04 — PORTS & PROTOCOLS
Ports are the doors; protocols are the languages. Master both and you understand how every application communicates over networks.
📖 What is a Port?

A port is a virtual communication endpoint, numbered 0–65535. If an IP address is the building address, the port is the apartment number. One server (one IP) can run dozens of services simultaneously because each uses a different port number.

Key Protocols Explained
🌐

HTTP/HTTPS

HyperText Transfer Protocol. HTTP=unencrypted (port 80), HTTPS=encrypted with TLS (port 443). All web browsing.

🔑

SSH

Secure Shell. Encrypted remote terminal access. Replaced Telnet. Uses port 22. Supports key-based auth.

📧

SMTP/IMAP/POP3

Email protocols. SMTP sends (port 25/587). IMAP retrieves without deleting (143/993). POP3 downloads & deletes (110/995).

🏠

DHCP

Assigns IP addresses automatically. DORA process: Discover, Offer, Request, Acknowledge. Ports 67/68.

📁

FTP/SFTP

File transfer. FTP (21) is plaintext — avoid. SFTP (22) is SSH-based and encrypted — use this.

📊

SNMP

Simple Network Management Protocol. Monitors network devices. Versions 1/2c use community strings (weak). Use SNMPv3.

What is a port number?
A port is a 16-bit number (0-65535) that identifies a specific process or service on a device. IP gets you to the right machine; the port number gets you to the right service running on that machine.
👆 Click to reveal answer
What is the difference between TCP and UDP?
TCP is connection-oriented with guaranteed, in-order delivery via three-way handshake — reliable but slower. UDP is connectionless with no delivery guarantees — faster but unreliable. TCP for HTTP/SSH; UDP for DNS/VoIP.
👆 Click to reveal answer
What port does SSH use and why is it a security concern?
SSH uses port 22. It's a security concern because it's a primary target for brute-force attacks. Best practice: change the default port, use key-based authentication only, restrict access by IP, and use fail2ban.
👆 Click to reveal answer
Why is port 23 (Telnet) dangerous?
Telnet transmits all data including usernames and passwords in plaintext — anyone sniffing the network can read them. It should never be used; SSH (port 22) is the secure replacement.
👆 Click to reveal answer
What is the significance of port 445 (SMB)?
SMB (Server Message Block) on port 445 was exploited by EternalBlue (NSA exploit leaked by Shadow Brokers), used in the WannaCry and NotPetya ransomware attacks. It should never be exposed to the internet.

05 — THE OSI MODEL
The 7-layer reference model that every security professional lives by. Every attack, defense tool, and protocol maps to a specific layer.
📖 Definition

The OSI (Open Systems Interconnection) Model is a conceptual framework developed by ISO that divides network communication into 7 distinct layers. Each layer has a specific job, talks only to the layers directly above/below it, and uses specific protocols. Knowing the OSI model means knowing exactly where an attack happens and what tools defend against it.

💡 Analogy — Sending a Package Internationally

Layer 7: You write the letter (your app creates data). Layer 6: Translate language (encoding). Layer 5: Open conversation with recipient. Layer 4: Choose delivery service (reliable FedEx = TCP, fast courier = UDP). Layer 3: Write the address (IP). Layer 2: Local post office handles it (MAC). Layer 1: The truck physically drives it (cables/Wi-Fi).

What is the OSI model and why does it matter?

What happens at Layer 4 (Transport)?
Layer 4 provides end-to-end communication between applications, handling segmentation, flow control, and error recovery. TCP and UDP operate here. Port numbers exist at Layer 4. SYN flood attacks target this layer.

What is encapsulation in networking?
Encapsulation is the process of each layer adding its own header (and sometimes trailer) to the data from the layer above as it travels down the stack. Decapsulation is the reverse — each layer strips its header going up.

Segment = Layer 4 unit (TCP/UDP + data). Packet = Layer 3 unit (IP header + segment). Frame = Layer 2 unit (MAC header + packet + FCS checksum). Bits = Layer 1 (raw binary on wire).

The TCP Three-Way Handshake
SYN

Client → Server: SYN (Synchronize)

Client sends a packet with SYN flag set and a random Initial Sequence Number (ISN). "I'd like to connect. My sequence starts at 1000."

S/A

When would you use UDP instead of TCP?
Use UDP when speed matters more than reliability: DNS queries (fast lookups, retried if lost), VoIP and video streaming (delay is worse than dropped frames), online gaming (low latency critical), and DHCP.
👆 Click to reveal answer
What is TCP flow control?
Flow control prevents the sender from overwhelming the receiver. TCP uses a 'window size' in headers — the receiver advertises how much buffer space it has. The sender can only send that many bytes before waiting for acknowledgement.
👆 Click to reveal answer
What is a TCP reset (RST)?
A TCP RST packet immediately terminates a connection without the normal FIN handshake. It can be legitimately sent when a connection is refused or invalid. Attackers use RST injection to forcibly terminate legitimate connections.

07b — TCP THREE-WAY HANDSHAKE
The most important handshake in networking. Every TCP connection starts here — and some of the most common attacks exploit exactly this process.
📖 Definition

The TCP Three-Way Handshake is the connection setup process used by TCP (Transmission Control Protocol) before any data can be exchanged between two devices. It ensures both sides are ready, synchronized, and agree on sequence numbers. Without completing this handshake, no TCP connection exists.

💡 Real World Analogy — Phone Call

Imagine calling a friend:
You: "Hello? Can you hear me?" (SYN — I want to connect)
Friend: "Yes I hear you! Can you hear me?" (SYN-ACK — I acknowledge you, and I'm ready too)
You: "Yes! Perfect." (ACK — confirmed, let's talk)
Now the conversation (data transfer) begins. That's exactly the TCP handshake.

Step-by-Step Breakdown
1

SYN — "I want to connect"

The client sends a TCP segment with the SYN (synchronize) flag set. It also picks a random Initial Sequence Number (ISN) — let's say 1000. This number is used to track packets in order.

Client → Server: [SYN] seq=1000

2

SYN-ACK — "Ready! Here's my info too"

The server replies with both SYN and ACK flags set. It acknowledges the client's sequence number (ACK = 1001, meaning "I got up to 1000, send me 1001 next") and picks its own random ISN — say 4000.

Server → Client: [SYN-ACK] seq=4000, ack=1001

3

ACK — "Got it. Let's go!"

The client acknowledges the server's sequence number (ACK = 4001). No data yet — just the final confirmation. The connection is now fully established (ESTABLISHED state) and data transfer can begin.

Client → Server: [ACK] ack=4001

How Attackers Exploit the Handshake — SYN Flood Attack
⚠️ SYN Flood — DoS Attack

The SYN flood attack exploits the fact that after receiving a SYN, the server allocates memory for a half-open connection and waits for the final ACK. If the ACK never comes, the server's connection table fills up and it can't accept any new legitimate connections — effectively a Denial of Service (DoS).

💡 Why Sequence Numbers Matter for Security

In the early internet, TCP sequence numbers were predictable. Attackers could guess the next sequence number and inject forged packets into a TCP session — called a TCP Session Hijacking attack. Modern OS implementations use cryptographically random ISNs to prevent this.

08 — DNS DEEP DIVE
The internet's phone book — and one of the most attacked protocols in existence. Understand DNS to understand a huge attack surface.
📖 Definition

DNS (Domain Name System) translates human-readable domain names (google.com) into machine-readable IP addresses (142.250.80.46). Without DNS, you'd need to memorize IP addresses for every website. DNS operates over port 53, primarily using UDP for queries.

DNS Resolution — Step by Step
1

Browser checks local cache

Before any network request, the browser checks if it already knows the IP (from a recent visit). Operating system also has a DNS cache (ipconfig /displaydns on Windows).

2

Ask the Recursive Resolver

If not cached, your device asks the configured DNS server (e.g., 8.8.8.8 Google, 1.1.1.1 Cloudflare, or your ISP's). The resolver does the heavy lifting.

3

Resolver asks Root Nameserver

13 sets of root nameservers globally. Resolver asks: "Who handles .com?" Root says: "Ask the .com TLD nameserver."

4

TLD Nameserver → Authoritative NS

.com TLD server says: "google.com's authoritative nameserver is ns1.google.com." Resolver then asks Google's own nameserver.

5

Authoritative Nameserver responds

Google's ns1.google.com answers: "142.250.80.46" — the real IP. Resolver caches it, sends it back. Browser connects. Entire process: ~20–100ms.

A SIEM (Security Information and Event Management) aggregates logs from all sources (firewall, IDS, DNS, endpoints), normalizes formats, correlates events across sources, and generates prioritized alerts. Examples: Splunk, Microsoft Sentinel, IBM QRadar.

What is NetFlow?
NetFlow is a protocol (originally Cisco) that exports metadata about network flows — source/destination IP, port, protocol, bytes, duration — without full packet content. Used for traffic analysis, anomaly detection, and capacity planning.

What logs should a SOC analyst check after a suspected breach?
Firewall logs (blocked/allowed connections), IDS/IPS alerts, DNS logs (suspicious domains), DHCP logs (which IP was assigned to what MAC), authentication logs (failed logins), proxy/web logs (URLs accessed), and endpoint logs.

1. Port Scanning
📖 What It Is

Port scanning sends packets to target ports to determine which are open (running services), closed (no service), or filtered (firewall). It's the first step in almost every network attack — attackers map what's available before deciding what to exploit.

NMAP — PORT SCANNING
# Basic TCP scan (top 1000 ports) nmap 192.168.1.1 # Detect service versions and OS nmap -sV -O 192.168.1.1 # Full scan all 65535 ports nmap -p- 192.168.1.1 # Stealth SYN scan (doesn't complete handshake) nmap -sS 192.168.1.1 # Scan entire subnet nmap -sn 192.168.1.0/24 # Ping sweep # Example output: PORT STATE SERVICE VERSION 22/tcp open ssh OpenSSH 8.2 80/tcp open http nginx 1.18 443/tcp open https 3306/tcp open mysql MySQL 8.0 ← Should NOT be exposed!
🔐 Defense

Host-based firewalls block scan attempts. IDS signatures detect Nmap scan patterns. Honeypots alert when scanned. Monitor logs for sequential port connection attempts from single source.

Interview Prep — Key Definitions

Click any card to reveal the answer. These are real interview questions for SOC analyst and networking roles.

What is a Man-in-the-Middle (MITM) attack?
An attacker secretly positions themselves between two communicating parties, intercepting and potentially modifying their traffic. Requires either network access (ARP spoofing) or a rogue access point. Defenses: HTTPS/TLS, HSTS, certificate pinning.
👆 Click to reveal answer
What is ARP spoofing?
An attacker sends unsolicited ARP replies claiming their MAC address is associated with a legitimate IP (usually the gateway). Victims update their ARP cache with the false mapping, routing their traffic through the attacker — enabling MITM.
👆 Click to reveal answer
What is a DDoS attack?
A Distributed Denial of Service attack uses multiple compromised systems (botnet) to flood a target with traffic, exhausting its resources and making it unavailable. Types: volumetric (bandwidth flood), protocol (SYN flood), application-layer (HTTP flood).
👆 Click to reveal answer
What is DNS cache poisoning?
An attacker injects forged DNS responses into a resolver's cache before the legitimate response arrives. Users querying that resolver are directed to attacker-controlled IPs. Prevention: DNSSEC, randomized source ports, query ID randomization.
👆 Click to reveal answer
What does Nmap do?
Nmap (Network Mapper) is a port scanner that discovers hosts and services on a network by sending packets and analyzing responses. It identifies open ports, running services, OS versions, and can run scripts for vulnerability detection.
